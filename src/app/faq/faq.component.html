How are reports stored?

link

Encrypted reports are stored in the browser's IndexedDB (local machine only by default, nothing is sent outside). Decryption keys are held in short-lived in-memory storage only — they are never written to sessionStorage, localStorage, or any other persistent medium.

There is an option to use an external database/server to store encrypted reports — see: API Reference, Video Tutorial.

Key lifetime & auto-lock

link

Decryption passwords are held exclusively in memory and are never persisted to any browser storage. The key vault is cleared automatically when any of the following occur:

  • visibility_off Tab hidden — switching away from the tab triggers an immediate clear
  • close Tab closed / page reload — the vault is wiped before the page unloads
  • hourglass_empty 15-minute inactivity — the timer resets on any keyboard, mouse, or touch input

After the vault clears, re-opening a report will prompt for the password again. This ensures that decryption keys are never left accessible in browser developer tools or recoverable from a disk dump.

Key Storage Security Level

link

VULNREPO offers two modes for holding decryption keys in the browser, configurable in Settings → Key Storage Security Level.

Secure
Memory only (Default)
Keys live exclusively in JavaScript memory and are never written to any browser storage. The vault is wiped automatically on tab switch, 15-minute idle, or page reload. Re-entering the security key after switching tabs is expected — it is a deliberate security measure to ensure that keys cannot be read from browser developer tools or recovered from disk.
Legacy
Session Storage (Lower security)
Keys are persisted in sessionStorage and survive tab switches and page reloads within the same browser session. They are cleared only when the tab is closed. Choose this mode if convenience is more important than strict key isolation, and you accept the trade-off that keys may be visible in browser developer tools while the session is open.

Why am I prompted for my key again after switching tabs?
You are using the default Secure — Memory only mode. Switching away from the tab clears the in-memory vault immediately. This prevents decryption keys from remaining accessible in an unattended or shared browser. If you find this disruptive, switch to Legacy — Session Storage in Settings, but be aware of the reduced security.

How to create a PDF report?

link
Option 1 — Simple

Download the HTML report, open it, and use the browser's Print to PDF feature (Ctrl+P). See: Video Tutorial • Chrome guide • Firefox guide

Option 2 — Advanced

Download the report as JSON and use the LaTeX generator: vulnrepo-json-to-latex-pdf. Build a custom LaTeX template and fill it with report data for full customisation.

How to restore an encrypted report if I forgot my security key?

link

Use a bruteforce method — or, ideally, remember the security key. There is no other way to decrypt the data.
