How are reports stored?

link

All data are stored in browser (by default on local machine, nothing is sent outside) using IndexedDB and SessionStorage.

There is an option to use an external database/server to store encrypted reports — see: API Reference, Video Tutorial.

How to create a PDF report?

link
Option 1 — Simple

Download the HTML report, open it, and use the browser's Print to PDF feature (Ctrl+P). See: Video Tutorial • Chrome guide • Firefox guide

Option 2 — Advanced

Download the report as JSON and use the LaTeX generator: vulnrepo-json-to-latex-pdf. Build a custom LaTeX template and fill it with report data for full customisation.

How to restore an encrypted report if I forgot my security key?

link

Use a bruteforce method — or, ideally, remember the security key. There is no other way to decrypt the data.

key

Can I change the report security key?

link

Yes — as long as you are able to decrypt the report with the current security key first.

How does displaying attachments in the HTML report work?

link

Images and .txt files are always displayed inline. All other file types are rendered as a download link.

What is the difference between .VULN and .VULNR?

link
.VULN Contains issues only
.VULNR Contains the full report — issues, metadata, author info, etc.

What is the issue status?

link

Issue status reflects the current state of review, fixing, or existence of an issue — visible to both researcher and in the final report.

  • report
    Open Waiting for review
  • warning
    Fix In Progress
  • check_circle
    Fixed
  • check_circle
    Won't Fix

How to add a private LLM?

link

Integrate with Ollama to run a local model.

  1. 1
    Install ollama.com
  2. 2
    Download and run a model:
    $ ollama run llama3.2:latest
    Browse available models.
  3. 3
    Allow additional web origins to access Ollama. If using vulnrepo.com, set: OLLAMA_ORIGINS=https://vulnrepo.com
